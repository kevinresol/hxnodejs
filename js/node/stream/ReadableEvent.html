import js.node.stream.Readable

Enumeration of events emitted by the Readable class.

Variables

inline read onlyClose:ReadableEvent<Void ‑> Void> = "close"

Emitted when the underlying resource (for example, the backing file descriptor) has been closed.

Not all streams will emit this.

inline read onlyData:ReadableEvent<EitherType<Buffer, String> ‑> Void> = "data"

If you attach a 'data' event listener, then it will switch the stream into flowing mode, and data will be passed to your handler as soon as it is available.

If you just want to get all the data out of the stream as fast as possible, this is the best way to do so.

inline read onlyEnd:ReadableEvent<Void ‑> Void> = "end"

This event fires when there will be no more data to read.

Note that the 'end' event will not fire unless the data is completely consumed. This can be done by switching into flowing mode, or by calling 'read' repeatedly until you get to the end.

inline read onlyError:ReadableEvent<Error ‑> Void> = "error"

Emitted if there was an error receiving data.

inline read onlyReadable:ReadableEvent<Void ‑> Void> = "readable"

When a chunk of data can be read from the stream, it will emit a readable event.

In some cases, listening for a readable event will cause some data to be read into the internal buffer from the underlying system, if it hadn't already.

Once the internal buffer is drained, a 'readable' event will fire again when more data is available.
