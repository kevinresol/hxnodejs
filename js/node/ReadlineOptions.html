package js.node

typedefReadlineOptions

import js.node.Readline

Properties

completer:Null<ReadlineCompleterCallback>

an optional function that is used for Tab autocompletion.

The completer function is given the current line entered by the user, and is supposed to return an Array with 2 entries:

* An Array with matching entries for the completion.
* The substring that was used for the matching.

Which ends up looking something like: [[substr1, substr2, ...], originalsubstring].

historySize:Null<Int>

maximum number of history lines retained. Defaults to 30.

input:IReadable

the readable stream to listen to

output:Null<IWritable>

the writable stream to write readline data to

terminal:Null<Bool>

pass true if the input and output streams should be treated like a TTY, and have ANSI/VT100 escape codes written to it.

Defaults to checking isTTY on the output stream upon instantiation.
